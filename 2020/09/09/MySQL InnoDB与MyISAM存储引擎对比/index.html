MySQL InnoDB与MyISAM存储引擎对比

发表于 2020-09-09 更新于 2020-10-16 分类于 MySQL
本文字数： 334 阅读时长 ≈ 1 分钟

InnoDB存储引擎

InnoDB引擎是Mysql的默认的存储引擎

  • 支持事务，InnoDB存储引擎主要就是为了在线事务处理（OLTP）的应用而设计的。
  • 行锁设计，支持外键，非锁定读。
  • 支持多版本的并发控制（MVCC）来获得高并发性。
  • 提供了插入缓冲、二次写、自适应哈希索引、预读等高性能和高可用的功能。

适用场景：

  • 需要事务的操作；
  • 更新数据需要使用行级锁；
  • 大数据量读写；
  • 大型互联网应用

MyISAM储存引擎介绍

  • 不支持事务，它的设计目标是面向在线分析的应用（OLAP）。
  • 支持全文索引。
  • 表锁设计。
  • 它的缓冲池只缓冲索引文件，不缓冲数据文件，所以 MyISAM 存储引擎表由 MYD 和 MYI 组成，前者存储数据文件，后者存储索引文件。

适用场景：

  • 不需要事务的操作；
  • 插入、更新少，读取频繁；
  • 频繁的统计计算
