102. 二叉树的层序遍历

给你一个二叉树，请你返回其按 层序遍历 得到的节点值。 （即逐层地，从左到右访问所有节点）。

示例：
二叉树：[3,9,20,null,null,15,7],

返回其层次遍历结果：

解题思路：

广度优先遍历：

  • 首先根元素入队
  • 当队列不为空的时候
    • 求当前队列的长度 s
    • 依次从队列中取 s个元素进行拓展，然后进入下一次迭代

深度优先遍历：

  • 首先根节点与层数0入栈
  • 当栈不为空的时候：
    • 获取当前层数
    • 如果结果集中不存在当前层，则添加
    • 添加元素到相应的层中
    • 从右向左添加孩子节点到栈中，然后进行下一次迭代

974.和可被 K 整除的子数组

令 P[i]=A[0]+A[1]+…+A[i]。那么每个连续子数组的和 sum(i,j) 就可以写成 P[j]−P[i−1]（其中0<i<j）的形式。此时，判断子数组的和能否被 K 整除就等价于判断 (P[j]−P[i−1])modK==0，根据 同余定理，只要 P[j]modK==P[i−1]modK，就可以保证上面的等式成立。

维护一个以前缀和模 K 的值为键，出现次数为值的哈希表 map，在遍历的同时进行更新。

贪心部分

所谓贪心算法是指，在对问题求解时，总是做出在当前看来是最好的选择。也就是说，不从整体最优上加以考虑，它所做出的仅仅是在某种意义上的局部最优解。
贪心算法没有固定的算法框架，算法设计的关键是贪心策略的选择。必须注意的是，贪心算法不是对所有问题都能得到整体最优解，选择的贪心策略必须具备无后效性（即某个状态以后的过程不会影响以前的状态，只与当前状态有关。）
所以，对所采用的贪心策略一定要仔细分析其是否满足无后效性。

二分查找

当我们要从一个序列中查找一个元素的时候，二分查找是一种非常快速的查找算法，二分查找又叫折半查找。它对要查找的序列有两个要求，一是该序列必须是有序的（即该序列中的所有元素都是按照大小关系排好序的，升序和降序都可以，本文假设是升序排列的），二是该序列必须是顺序存储的。

  • 如果目标值等于中间元素，则找到目标值。
  • 如果目标值较小，继续在左侧搜索。
  • 如果目标值较大，则继续在右侧搜索。

查找第一个大于等于目标值的元素位置

  1. mid = left + (right - mid) / 2
  2. 如果 target > nums[mid]，说明插入位置在右边，left = mid + 1;
  3. 如果 target <= nums[mid], 分为两种情况：
    1. 如果 target < nums[mid]，说明插入位置在左边，right = mid - 1;
    2. 如果 target == nums[mid]，此时，需要查看mid 左边的元素是否也等于target，仍然right = mid - 1;
  4. 退出条件为 mid == left == right;
    1. 如果 target > nums[mid]，left = mid + 1 > right，left 位置即为插入位置
    2. 如果 target < nums[mid]，right = mid - 1 < left，left 位置即为插入位置
    3. 如果 target == nums[mid]，那么第一个该元素肯定是target在数组中的第一个位置
